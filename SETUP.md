# Setup & Deployment Guide - Aviator Predictor\n\n## 🚀 Quick Local Setup (Development)\n\n### Prerequisites\n- Node.js 18+\n- Python 3.11+\n- Docker & Docker Compose (optional but recommended)\n\n### Step 1: Clone the Repository\n\n```bash\ngit clone https://github.com/tanatswajiri0-arch/aviator-predictor1.git\ncd aviator-predictor1\n```\n\n### Step 2: Environment Configuration\n\n```bash\ncp .env.example .env\n```\n\nEdit `.env` and add your Spin City API credentials:\n\n```bash\nSPIN_CITY_API_KEY=your_api_key_here\nSPIN_CITY_API_URL=https://api.spincity.com\nSPIN_CITY_WS_URL=wss://ws.spincity.com\n```\n\n### Step 3: Setup Backend\n\n```bash\ncd backend\nnpm install\nnpm start\n```\n\n✅ Backend runs on `http://localhost:5000`\n\n### Step 4: Setup ML Service (New Terminal)\n\n```bash\ncd ml-service\npip install -r requirements.txt\npython app.py\n```\n\n✅ ML Service runs on `http://localhost:5001`\n\n### Step 5: Setup Frontend (New Terminal)\n\n```bash\ncd frontend\nnpm install\nnpm start\n```\n\n✅ Frontend runs on `http://localhost:3000`\n\n---\n\n## 🐳 Docker Setup (Recommended)\n\n### Prerequisites\n- Docker\n- Docker Compose\n\n### Step 1: Clone & Configure\n\n```bash\ngit clone https://github.com/tanatswajiri0-arch/aviator-predictor1.git\ncd aviator-predictor1\ncp .env.example .env\n```\n\nEdit `.env` with your API credentials\n\n### Step 2: Build & Run\n\n```bash\ndocker-compose up -d\n```\n\n### Step 3: Verify Services\n\n```bash\n# Check running containers\ndocker-compose ps\n\n# View logs\ndocker-compose logs -f backend\ndocker-compose logs -f ml-service\ndocker-compose logs -f frontend\n```\n\n### Access Your Application\n\n- 🌐 **Frontend**: http://localhost:3000\n- 🔌 **Backend API**: http://localhost:5000\n- 🤖 **ML Service**: http://localhost:5001\n\n### Stop Services\n\n```bash\ndocker-compose down\n```\n\n---\n\n## ☁️ Deployment Options\n\n### Option 1: Azure App Service (Recommended)\n\n#### Prerequisites\n- Azure Account\n- Azure CLI\n\n#### Step 1: Create Azure Resources\n\n```bash\n# Create resource group\naz group create --name aviator-rg --location eastus\n\n# Create App Service Plan (Linux)\naz appservice plan create \\\n  --name aviator-plan \\\n  --resource-group aviator-rg \\\n  --is-linux \\\n  --sku B1\n\n# Create web app for backend (Node.js)\naz webapp create \\\n  --name aviator-backend \\\n  --plan aviator-plan \\\n  --resource-group aviator-rg \\\n  --runtime \"NODE|18-lts\"\n\n# Create web app for frontend (Node.js)\naz webapp create \\\n  --name aviator-frontend \\\n  --plan aviator-plan \\\n  --resource-group aviator-rg \\\n  --runtime \"NODE|18-lts\"\n\n# Create web app for ML service (Python)\naz webapp create \\\n  --name aviator-ml-service \\\n  --plan aviator-plan \\\n  --resource-group aviator-rg \\\n  --runtime \"PYTHON|3.11\"\n```\n\n#### Step 2: Configure Deployment Credentials\n\n```bash\n# Get publish profile for backend\naz webapp deployment publish-profile \\\n  --name aviator-backend \\\n  --resource-group aviator-rg \\\n  --xml > backend-profile.xml\n\n# Get publish profile for frontend\naz webapp deployment publish-profile \\\n  --name aviator-frontend \\\n  --resource-group aviator-rg \\\n  --xml > frontend-profile.xml\n\n# Get publish profile for ML service\naz webapp deployment publish-profile \\\n  --name aviator-ml-service \\\n  --resource-group aviator-rg \\\n  --xml > ml-profile.xml\n```\n\n#### Step 3: Add GitHub Secrets\n\n1. Go to GitHub Repository → Settings → Secrets and variables → Actions\n2. Add the following secrets:\n   - `AZURE_BACKEND_PUBLISH_PROFILE` - (contents of backend-profile.xml)\n   - `AZURE_FRONTEND_PUBLISH_PROFILE` - (contents of frontend-profile.xml)\n   - `AZURE_ML_PUBLISH_PROFILE` - (contents of ml-profile.xml)\n\n#### Step 4: Enable Deployment\n\nThe GitHub Actions workflow (`.github/workflows/ci.yml`) will automatically deploy when you push to main.\n\n---\n\n### Option 2: Heroku Deployment\n\n#### Prerequisites\n- Heroku Account\n- Heroku CLI\n\n#### Step 1: Login to Heroku\n\n```bash\nheroku login\n```\n\n#### Step 2: Create Apps\n\n```bash\n# Backend\nheroku create aviator-backend --buildpack heroku/nodejs\n\n# Frontend\nheroku create aviator-frontend --buildpack heroku/nodejs\n\n# ML Service\nheroku create aviator-ml --buildpack heroku/python\n```\n\n#### Step 3: Set Environment Variables\n\n```bash\nheroku config:set SPIN_CITY_API_KEY=your_key -a aviator-backend\nheroku config:set ML_SERVICE_URL=https://aviator-ml.herokuapp.com -a aviator-backend\n```\n\n#### Step 4: Deploy\n\n```bash\n# Backend\ncd backend\ngit push heroku main\n\n# Frontend\ncd ../frontend\ngit push heroku main\n\n# ML Service\ncd ../ml-service\ngit push heroku main\n```\n\n---\n\n### Option 3: Docker Hub + Any VPS\n\n#### Step 1: Build Docker Images\n\n```bash\n# Login to Docker Hub\ndocker login\n\n# Build backend\ndocker build -t yourusername/aviator-backend:latest ./backend\ndocker push yourusername/aviator-backend:latest\n\n# Build frontend\ndocker build -t yourusername/aviator-frontend:latest ./frontend\ndocker push yourusername/aviator-frontend:latest\n\n# Build ML service\ndocker build -t yourusername/aviator-ml:latest ./ml-service\ndocker push yourusername/aviator-ml:latest\n```\n\n#### Step 2: Deploy on VPS (DigitalOcean, AWS, etc.)\n\n```bash\n# SSH into your server\nssh user@your-vps-ip\n\n# Install Docker\ncurl -fsSL https://get.docker.com -o get-docker.sh\nsudo sh get-docker.sh\n\n# Create docker-compose file and run\ndocker pull yourusername/aviator-backend:latest\ndocker pull yourusername/aviator-frontend:latest\ndocker pull yourusername/aviator-ml:latest\n\ndocker run -d -p 5000:5000 --name backend yourusername/aviator-backend:latest\ndocker run -d -p 5001:5001 --name ml-service yourusername/aviator-ml:latest\ndocker run -d -p 3000:3000 --name frontend yourusername/aviator-frontend:latest\n```\n\n---\n\n## 🧪 Testing\n\n### Test Backend\n\n```bash\ncurl http://localhost:5000/health\n```\n\nExpected logs -f service-name\n```\n\n### Monitor on Production\n\n**Azure**: Use Azure Portal or Azure CLI\n```bash\naz webapp log tail --name aviator-backend --resource-group aviator-rg\n```\n\n**Heroku**: file:\n```bash\nML_SERVICE_URL=http://ml-service:5001  # Docker\nML_SERVICE_URL=http://localhost:5001   # Local\n```\n\n### CORS errors\n\nEnsure CORS is enabled in backend/server.js:\n```javascript\napp.use(cors());\n```\n\n### Port already in use\n\n```bash\n# Find process using port\nlsof -i :5000\n\n# Kill process\nkill -9 <PID>\n```\n\n### Docker issues\n\n```bash\n# Rebuild images\ndocker-compose build --no-cache\n\n# Remove old containers\ndocker-compose down -v\ndocker-compose up -d\n```\n\n---\n\n## ✅ Deployment Checklist\n\n- [ ] Clone
